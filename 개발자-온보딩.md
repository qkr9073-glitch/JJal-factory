# 짤공장 개발자 온보딩 (다른 컴퓨터에서 기여하기)

> 이 문서는 **사장님 PC가 아닌 다른 컴퓨터에서** 짤공장에 새 기능·수정을 넣는 개발자(그리고 그 개발자의 클로드코드)를 위한 안내입니다.
> 프로젝트 전반 구조는 [`CLAUDE.md`](CLAUDE.md)를, 최종 사용자(운영자) 사용법은 [`README.md`](README.md)를 보세요.

---

## 0. 딱 한 장 요약

- **운영(뽑기·검수·업로드)** 은 코드 없이 브라우저만 있으면 됨 → 공개 주소 접속(§6).
- **개발(신기능·수정)** 은 이 문서대로: 클론 → 코드 → 테스트 → `git push` → **사장님이 검토 후 `배포.cmd`** 로 반영.
- 실제 서버·인스타 계정·크롤링은 **사장님 PC에서만** 돈다. 내 컴퓨터는 코드만 만든다.
- 🚨 **인스타 업로드 코드를 함부로 실행하면 사장님 실계정에 진짜 올라간다.** §5를 반드시 읽을 것.

---

## 1. 큰 그림 — "코드 짜는 곳"과 "서버 도는 곳"은 다르다

```
[내 컴퓨터]                              [사장님 PC = 유일한 서버]
  나 ↔ 내 클로드코드                        · 실제 크롤링 (집 IP라야 됨)
  · 코드 작성 / 로컬 테스트                  · 인스타 실계정 토큰 보관
  · git push  ───────── GitHub ─────────▶  · 사장님이 배포.cmd → 라이브 반영
       ▲                                          │
       └────────── 문제 생기면 git 롤백 ◀──────────┘
```

- 크롤링은 **사장님 집 IP** 라야 뚫리는 사이트가 있어서(디시 등), 실제 수집·게시는 사장님 PC에서만.
- 내 컴퓨터에서 하는 건 **코드 작성과 로직 테스트**뿐. 그건 IP·계정 상관없이 어디서든 된다.

## 2. 처음 1번 세팅

1. **클론**
   ```
   git clone <저장소_주소>  meme-factory
   cd meme-factory
   ```
   ※ 폴더 위치는 자유(예: `D:\meme-factory`). C: 드라이브는 피할 것.

2. **파이썬 준비** (Python 3.12 권장)
   ```
   python -m venv .venv
   .venv\Scripts\activate
   pip install -r requirements.txt
   ```

3. **설정 파일 만들기** — `config.json` 은 저장소에 없다(비밀키라 제외됨).
   ```
   copy config.example.json config.json
   ```
   그리고 `config.json` 을 열어 최소한 이것만 채운다:
   - `gemini_api_key` — https://aistudio.google.com/apikey 에서 **본인 키** 발급(무료). 비우면 `--mock` 만 됨.
   - `access_code` — 로컬 접속 암호(아무거나).
   - (선택) `pexels_key`, 인스타 항목 등은 로컬 개발엔 비워도 됨.
   - `card_profile_photo` 는 사장님 프로필 사진 경로라 내겐 없다 → 비워두면 된다.
   > ⚠️ `config.json` 저장 인코딩은 **BOM 없는 UTF-8**. §4 참고.

4. **로컬 실행**
   ```
   python server.py
   ```
   브라우저에서 http://localhost:8777/ (짤공장), http://localhost:8777/card (카드뉴스).

## 3. 개발 → 반영 흐름 (매번)

```
1) git pull                     # 최신 코드부터 받고 시작
2) git switch -c feat/기능이름    # 새 브랜치에서 작업
3) (코드 작성 + 로컬 테스트)       # 클로드코드랑 자유롭게 반복
4) git add -A && git commit -m "무엇을 왜 바꿨는지"
5) git push -u origin feat/기능이름
6) 사장님에게 알림 → 사장님이 검토 후 `배포.cmd` 더블클릭 → 라이브
```

- 커밋 메시지는 **한국어로 또렷하게**. 사장님 쪽 클로드코드(=검토자)가 이 메시지로 맥락을 잡는다.
- 되도록 `main` 에 직접 커밋하지 말고 브랜치로. 사고나도 롤백이 쉽다.

## 4. 🚨 이 PC/프로젝트의 절대 규칙 (지뢰밭)

| 규칙 | 이유 |
|---|---|
| **`config.json` 은 BOM 없는 UTF-8 로만 저장** | PowerShell `Out-File` 등으로 저장하면 BOM이 붙어 `json.loads` 가 깨져 **서버 전체가 죽는다**. 클로드코드로 고칠 땐 **Edit 툴**로. |
| **`config.json` 은 커밋 금지** (이미 `.gitignore` 됨) | 인스타 토큰·API키 뭉치. 유출 = 계정 탈취. |
| **한글 들어간 `.ps1` 은 UTF-8 *BOM 필수*** (config와 반대!) | BOM 없으면 파싱이 깨진다. |
| **`.cmd` / `.vbs` 주석은 영문만** | CP949 콘솔에서 한글 주석이 인코딩 깨짐. |
| **파이썬 `print` 에 특수문자(—, 이모지 등) 조심** | CP949 콘솔에서 `UnicodeEncodeError`. `PYTHONUTF8=1` 로 실행 권장. |
| **`결과물/` 폴더명·`output_dir` 함부로 바꾸지 말 것** | server.py·pipeline 여러 곳이 같은 값을 읽는다. |
| **실제 완성팩이 아니라 복사본에서 테스트** | 원본 팩을 망가뜨리면 되돌리기 어렵다. |

## 5. 테스트 요령 — 특히 "실제 게시" 주의

- **인스타 업로드/릴스 게시 함수(`insta.py`, `/api/insta/*`, `/api/reel/*`)를 테스트로 실행하면
  사장님 실계정에 진짜로 올라간다.** 로컬 `config.json` 의 인스타 항목을 비워두면 애초에 발행이 안 되니 안전.
  실제 게시 동작 검증은 **사장님과 함께** 사장님 PC에서 한다.
- Gemini 없이 파이프라인만 볼 땐 `--mock`:
  ```
  python make.py <URL> --mock --no-open
  python make_card.py "주제" --mock --no-open
  ```
- **크롤링**은 집 IP라야 되는 사이트가 있어 로컬에선 실패할 수 있다. 저장된 결과물/샘플 HTML로 렌더·로직만 검증하고, 실제 크롤 검증은 사장님 PC에서.
- 서버 재시작은 사장님이 `배포.cmd`(또는 `RESTART-SERVER.cmd`)로. **내가 사장님 서버를 직접 만지지 않는다.**

## 6. 코드 없이 "운영만" 할 거면 (개발자 아닌 직원)

짤공장은 웹앱이라, **개발 없이 사장님 역할(뽑기·검수·업로드)** 은 브라우저만으로 가능:

1. 공개 주소 접속(사장님에게 URL 받기).
2. 접속코드(`access_code`) 입력.
3. 인기글 불러오기 → 만들기 → 검수 → 📤 업로드. **결과·업로드는 전부 사장님 계정으로** 나간다(서버가 사장님 PC라서).

즉 운영은 세팅 0. 이 문서 §2~§5는 "새 기능을 만들" 때만 필요하다.

## 7. 막히면

- 로컬 서버가 안 뜨면: `pip install -r requirements.txt` 다시, 포트 8777 사용 중인지 확인.
- 카드뉴스/짤 상세 오류: `python make.py <URL> --debug`.
- 구조가 헷갈리면 [`CLAUDE.md`](CLAUDE.md)에 파일별 역할이 정리돼 있다(클로드코드가 자동으로 읽음).
- 그 외는 사장님에게. 라이브 반영·실계정 관련 판단은 사장님 몫.
