# 작업자 인수인계 — 여기부터 읽으세요 (START HERE)

> 다른 PC에서 이 프로젝트(짤공장)에 합류하는 작업자를 위한 문서입니다.
> **기술 설치·기여 방법**은 [`개발자-온보딩.md`](개발자-온보딩.md), **프로젝트 상세 구조**는 [`CLAUDE.md`](CLAUDE.md) 에 있습니다.
> 클로드코드를 쓰면 저 두 파일을 자동으로 읽어서 프로젝트를 바로 이해합니다.

---

## 1. 이 프로젝트가 뭐냐

인스타그램 **콘텐츠 자동 생성 공장** 웹앱입니다. 4가지를 만듭니다:
- **짤공장** — 커뮤니티 유머글(디시·루리웹·에펨 등) → 썸네일+짤+캡션 완성팩
- **카드뉴스** — 주제 한 줄 → 4:5 정보형 카드뉴스 + 전자책 PDF (테마 6종)
- **스토리** — 사연 → 서사형 카드 (일본 계정 `kangaroostory.jp` = 일본어 자동)
- **릴스** — MP4 영상 → 인스타 릴스 자동 게시 (+ AI가 영상 보고 캡션 생성)

만든 결과물은 웹에서 검수 후 **인스타에 자동 업로드**됩니다. 두뇌는 **Gemini 2.5 Flash**.

## 2. 큰 그림 — 제일 중요 ⭐

**"코드 짜는 곳"과 "서버 도는 곳"은 다릅니다.**

```
[당신 PC]                              [사장님 PC = 유일한 서버]
  코드 작성 / 로컬 테스트                  · 실제 크롤링 (집 IP라야 됨)
  git push (main)  ──── GitHub ───────▶  · 인스타 실계정 토큰 보관
       ▲                                 · push되면 ~1분 내 자동으로 라이브 반영
       └───── 문제 생기면 git 롤백 ◀────────┘
```

- 실제 크롤링·인스타 게시·서버는 **사장님 PC에서만** 돕니다. 당신은 **코드만** 만듭니다.
- **당신이 main에 push하면 ~1분 뒤 자동으로 라이브에 반영**됩니다(자동배포). 사장님이 뭘 누를 필요 없음.

## 3. 당신이 하는 일

- 새 기능 개발, 버그 수정, 개선.
- 자기 PC에서 클로드코드랑 코드 짜고 → 로컬에서 테스트 → `git push` → 자동 반영.

## 4. 시작하기 (자세한 건 개발자-온보딩.md)

1. **깃허브 초대 수락** — 사장님이 이 저장소 collaborator로 초대합니다. 이메일/알림에서 수락.
2. **클론** → `git clone <저장소주소> meme-factory` (D:\ 등, C: 는 피할 것)
3. **파이썬 세팅** → `pip install -r requirements.txt` (Python 3.12)
4. **설정 파일** → `copy config.example.json config.json` 후 본인 Gemini 키만 채우면 로컬 실행 OK
5. **실행** → `python server.py` → http://localhost:8777/

## 5. ⚠️ 절대 규칙 (지뢰밭 — 꼭 지키기)

| 규칙 | 이유 |
|---|---|
| **`config.json` 은 BOM 없는 UTF-8 로만 저장** | BOM 붙으면 서버 전체가 죽음. 클로드코드로 고칠 땐 Edit 툴로. |
| **`config.json` 커밋 금지** (이미 .gitignore) | API 키·토큰 뭉치. 유출 = 계정 탈취. |
| **`.py` 파일 UTF-8 / `.cmd` 주석은 영문만** | CP949 콘솔 인코딩 문제. |
| **push 전 반드시 로컬 테스트** | main에 push하면 검토 없이 ~1분 뒤 바로 라이브다. |
| 🚨 **인스타 게시/릴스 함수를 테스트로 실행하면 사장님 실계정에 진짜 올라감** | 로컬 config의 인스타 항목을 비워두면 안전. 실제 게시 검증은 사장님과 함께. |

## 6. 작업 흐름 (매번)

```
git pull                     # 최신부터
(코드 작성 + 로컬 테스트)      # push 전 꼭 확인
git add -A && git commit -m "무엇을 왜 바꿨는지 (한국어로 또렷하게)"
git push                     # main에 push → 약 1분 뒤 자동 라이브!
```

작업 중(WIP)은 브랜치에서 하다가, **완성돼서 라이브 낼 준비가 됐을 때만 main에 머지/푸시**하세요.

## 7. 운영도 할 수 있습니다 (코드 없이)

이건 웹앱이라 개발 없이도 **뽑기·검수·인스타 업로드**를 브라우저만으로 할 수 있어요.
사장님에게 **공개 주소 + 접속코드**를 받아 접속 → 인기글 불러오기 → 만들기 → 📤 업로드.
(결과·업로드는 전부 사장님 계정으로 나갑니다. 서버가 사장님 PC라서.)

## 8. 클로드코드 쓰면 편합니다

당신도 클로드코드를 쓰면, 클론받은 순간 클로드가 [`CLAUDE.md`](CLAUDE.md)·[`개발자-온보딩.md`](개발자-온보딩.md)를 읽고
**이 프로젝트의 구조·규칙·함정을 그대로 파악**합니다. "이 기능 추가해줘" 하면 규칙 지키며 작업합니다.
