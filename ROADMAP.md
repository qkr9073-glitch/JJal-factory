# 짤공장 v2 개편 ROADMAP

> 2026-07 확정 기획. PC 가로형 UI 전면 개편 + 신기능. 착수 순서: **① 에메랄드 색 → ② 유튜브→짤(먼저 검증) → ③ 계정/권한 + 예약 스케줄러 → ④ 수입·수출 → ⑤ 커뮤니티 추가**.
> 이 문서는 기획 합의본이다. 세부 UI는 구현하며 조정.

---

## 0. 전제 (안 바뀌는 원칙)
- **서버 = 사장님 PC 하나.** 일반 사용자는 브라우저 접속만. 생성·크롤·게시·저장 전부 서버에서.
- 팩은 `결과물/`(서버 디스크)에 저장. 사용자는 zip으로 로컬 소장 가능.
- `main` push → ~1분 뒤 자동 라이브. 완성된 것만 push, 작업은 브랜치.
- `config.json` BOM 없는 UTF-8·Edit로만·커밋 금지. 인스타 항목 로컬은 비움.

---

## 1. 색 · 레이아웃
- 액센트 **에메랄드** (다크/라이트 공통). 라이트의 웜골드(똥색) 제거, 살짝 초록기 도는 뉴트럴.
- **PC 가로형** 레이아웃 기준(거의 모든 작업이 PC). 폰 대응은 유지.

## 2. 상단 탭 구조
```
제작소(짤 · 카드뉴스)  |  수입·수출  |  릴스  |  결과물(내 것만)  |  예약·승인  |  관리자(관리자만)
```
- **제작소**: 짤 / 카드뉴스 서브탭.
- **결과물**: 내가 만든 것만. (관리자도 자기 것만; 전원 결과물은 예약·승인에서)
- **릴스**: 완성 영상(MP4) 업로드·예약 전용 독립 탭.

## 3. 인증 · 권한 (경량 · DB 없음)
- **4자리 숫자 회원코드**(비번 없음). JSON 파일 저장(gitignore). 틀린 코드 rate-limit.
- 가입 = 공개폼에 **이름 + 관리자 메모** → 관리자 "가입승인" → **코드 발급**. 이후 코드로 입장(브라우저 저장).
- **관리자**(동등 권한): 전 기능 + **즉시 업로드** + 예약(자동승인). 관리자 메뉴 = 가입승인 / 인스타 계정추가 / 업로드 승인 / 멤버 관리(삭제·관리자 승격) / **API 키 관리**.
- **일반**: **예약 신청만**(즉시 업로드 불가, 승인 필요). 전원 예약 스케줄 열람 가능.

## 4. API 키 관리 (범용 레지스트리)
- 키를 계속 **추가 가능**한 목록(이름/제공자/키/공유·개인).
- **공유**(관리자 결제, 한 명 수정=전원 반영): Gemini, Pexels. 앞으로 Codex 등 추가.
- **개인별**: YouTube Data API.

## 5. 예약 · 스케줄러
- **신설 필요**: 사장님 PC 상시 가동되는 스케줄러가 예약 큐를 주기 확인 → 시간 되면 자동 게시.
- 예약 시각에 **PC 꺼져 있었으면 → 건너뛰고 "놓침" 표시** (밀린 것 자동 게시 안 함).
- **예약·승인 = 캘린더**: 제작자 + 짤/카드/릴스 미리보기(hover: 영상 자동재생 / 짤 자동 슬라이드). **일정 겹치면 강조 표시만**, 판단은 사람.
- 일반 사용자도 **남의 예약까지 열람**(겹침 방지). 예약된 릴스 MP4는 **게시 완료까지 보관**.

## 6. 유튜브 → 짤 (짤 탭 안 '유튜브 모드') ★ 먼저 착수
결과물이 짤 성격이라 **짤로 분류**. 짤 탭에서 소스만 전환.

### 파이프라인 (목표 ≤5분)
1. URL 붙이면 **oEmbed로 썸네일·제목 즉시** 표시 → 대표 썸네일 템플릿부터 선택.
2. 병렬: (A) **Gemini가 대본 + 뒷장 구성안** 생성 / (B) **yt-dlp 1080p 다운로드**.
3. **ffmpeg로 프레임 풀 20~30장 추출** + **선명도 자동 필터**(흐린/어두운 컷 제외). **원본 영상은 추출 후 삭제.**
4. 뒷장별 프레임 **자동 배정**(그 장 대본 구간 근처) + **🎲 리롤**(풀에서 즉시).
5. 확정 → 템플릿 렌더 → 완성팩. **다운로드 실패 시 템플릿 생성 이미지로 폴백**.
- 뒷장 개수 = 대본 길이에 맞춰 자동. 화질 위해 1080p(360p 금지).

### 키
- **URL→짤 핵심은 새 키 0개** (yt-dlp + 기존 Gemini). → **먼저 이 경로부터 검증.**
- YouTube Data API 키는 **인기 목록 검색에만** 사용(개인별).

### '이슈' 패널 (짤 탭 우측)
```
[이슈]
 ├ ▶️ 유튜브   (맨 앞, 기본)
 └ 🗨 커뮤니티 → [커뮤니티 전체 | 디시 | 루리웹 | 에펨 | +계정별 추가사이트]
```
- 유튜브 트렌딩 기본 = **한국·미국·일본 각 탑10**. 검색 **30/50/100** 선택. 한국어 검색어 → **영/일 번역 검색**(해외 쇼츠 → 한국 짤).
- 트렌딩 **일일 자동 갱신**(스케줄러) + **수동 새로고침 버튼**.

## 7. 팩 원천 데이터 저장 (수입·수출 대비)
- 모든 제작소 팩에 **`source.json`** 저장: 대본 / 장별 텍스트 / 후킹 / 캡션 / 언어 / **채택 프레임** / 썸네일 / 소유자 코드.
- **프레임 보관 정책**: 원본 영상 삭제, **프레임 풀은 유지**(리롤 아무때나 + 재변환 즉시). 팩당 ~15~20MB. 공간 빡세지면 "오래된 팩 프레임 정리" 버튼.
- 옛 팩(데이터 없음)은 OCR 폴백.

## 8. 수입·수출 (짤 + 카드뉴스, 릴스 제외 · '일본 이관' 흡수)
- **시작 언어 → 도착 언어** 선택. 시작: **KO / JA / EN** (구조는 언어 추가 쉽게).
- 소스: 새 이미지 업로드 **또는 결과물 팩 가져오기**(source.json으로 OCR 없이 재번역).
- 카드뉴스: **카드만 기본 렌더**, **전자책은 선택**.
- 도착 언어 폰트 필요(KO/JA/EN 보유). RTL(아랍어 등)은 추후 별도.

## 9. 스타일 프리셋(내용) vs 템플릿(디자인) — 명확 구분
- **🧩 스타일 프리셋 = 내용 구성**: 레퍼런스(주로 이미지, txt도) 업로드 → **OCR로 구성/흐름/톤 학습**.
- **🎨 템플릿 = 디자인**: 레퍼런스 이미지 → **색/폰트/레이아웃 학습**. 기존 6종 + `＋`(새 학습, 이름 지정).
- UI: 업로드 칸 분리 + "**내용을 베낄지 / 디자인을 베낄지**" 라벨로 초보도 구분.

## 10. 커뮤니티 추가 (마지막)
- 커뮤 유머게시판 링크 넣으면 인식해 대표 인기글 수집. **계정별 저장 = 그 사람에게만 보임**.
- 주의: 크롤은 사장님 PC IP에서 → 로그인월/차단 사이트는 실패 가능(감수). 임의 사이트는 범용 파서라 실패 여지.
